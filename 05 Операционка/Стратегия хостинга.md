# Стратегия хостинга

> Навигация: [[Индекс Obsidian]] · [[Навигация по операционке]]
>
> Связанные документы: [[Единая продуктовая спецификация]] · [[Чеклист запуска]] · [[Бизнес-модель]]

## Контекст
Текущий бот работает как:
- Python + aiogram
- `start_polling`
- SQLite (`bot.db`)
- always-on процесс

Это значит:
- нужен постоянный процесс;
- нужен постоянный диск;
- serverless и "засыпающие" решения подходят плохо;
- для стабильности лучше обычный VPS или worker-платформа с volume.

## Главный вывод
Если приоритет:
- минимальные траты;
- высокая производительность;
- предсказуемая работа;

то базовая рекомендация такая:

### Для demo
**Hetzner Cloud CX23** как основной вариант.

### Для клиентских ботов
**Hetzner Cloud CX23** по умолчанию,
**CX33** для более тяжёлых клиентов или нескольких ботов на одном сервере.

## Почему не ставить всё на serverless
Для этого проекта serverless неидеален, потому что:
- бот работает через polling;
- SQLite требует локального постоянного хранения;
- scheduler и фоновые задачи должны жить постоянно;
- "просыпающиеся" платформы ухудшают UX и надёжность.

## Варианты

### 1. Hetzner Cloud
Подходит лучше всего по соотношению цена / мощность.

Сильные стороны:
- дешёвый VPS;
- заметно лучше по ресурсам, чем типичные starter-планы PaaS;
- полный контроль;
- легко держать polling + SQLite + systemd;
- удобно для demo и для боевых клиентских ботов.

Когда выбирать:
- если хочешь минимальную себестоимость;
- если готов один раз настроить сервер;
- если важна максимальная отдача за деньги.

### 2. Railway
Подходит как компромисс между удобством и ценой.

Сильные стороны:
- очень быстро стартовать;
- удобный deploy;
- есть volumes;
- меньше ручной инфраструктуры.

Слабые стороны:
- дороже VPS при долгой работе always-on бота;
- volume накладывает ограничения на реплики;
- по цене/производительности слабее Hetzner.

Когда выбирать:
- если важнее удобство, чем минимальная себестоимость;
- если нужен быстрый запуск demo без желания возиться с сервером.

### 3. Fly.io
Подходит для тех, кто хочет дешёвый always-on runtime, но готов мириться с большей инфраструктурной сложностью.

Сильные стороны:
- низкий порог по цене для маленькой машины;
- volumes;
- always-on модель.

Слабые стороны:
- сложнее в сопровождении, чем Railway;
- volume локален для конкретной машины;
- snapshots по умолчанию не заменяют нормальный backup-процесс.

Когда выбирать:
- если комфортно жить в более инженерном хостинге;
- если хочется дёшево, но не хочется классический VPS.

### 4. Render
Для этого проекта не основной выбор.

Почему:
- background worker стоит дороже, чем Hetzner/Fly для такого бота;
- для polling + SQLite это не даёт выигрыша по экономике;
- удобен, но не лучший по cost/performance.

## Рекомендация по ролям

### Demo-бот
**Основной выбор: один отдельный Hetzner CX23**

Зачем:
- demo должен быть всегда доступен;
- на нём не страшно держать SQLite;
- дёшево и мощно;
- легко перезапускать, логировать и обслуживать.

### Первый клиент
**Основной выбор: отдельный Hetzner CX23**

Зачем:
- изоляция клиента;
- простая поддержка;
- легко считать себестоимость;
- не смешивается с demo.

### Несколько первых клиентов при экономии
Если нужно ужаться по расходам, допустим вариант:
- один сервер для demo;
- один сервер для 2-5 клиентских ботов.

Но это уже компромисс:
- хуже изоляция;
- сложнее сопровождение;
- выше риск, что один бот влияет на другой.

### Для более тяжёлого клиента
Если:
- больше записей;
- больше фото;
- больше фоновых задач;
- несколько ботов на одном сервере;

тогда идти в **Hetzner CX33**.

## Практическая стратегия

### Самый здоровый вариант сейчас
1. Demo держать отдельно.
2. Первого клиента держать отдельно.
3. Для новых клиентов по умолчанию использовать отдельный VPS.

Это чуть менее экономно на бумаге, но:
- проще считать;
- проще переносить;
- проще поддерживать;
- меньше хаоса.

### Самый дешёвый вариант
1. Demo + первые клиенты на одном Hetzner CX33.
2. Позже разносить по отдельным серверам.

Это дешевле, но менее аккуратно.

## Моя итоговая рекомендация

### Если нужен лучший баланс
- **Demo:** Hetzner CX23
- **Клиент:** Hetzner CX23
- **Рост / несколько ботов:** Hetzner CX33

### Если важнее удобство, чем минимальная цена
- **Demo:** Railway Hobby + volume
- **Клиент:** Hetzner CX23

## Что важно помнить
SQLite здесь нормален для demo и первых клиентов, пока бот работает в одном процессе и на одном сервере.

Если позже появятся:
- несколько инстансов;
- вебхуки с масштабированием;
- заметный рост нагрузки;
- сильная многопользовательская работа;

тогда следующий шаг — переход с SQLite на Postgres.
