# Единая продуктовая спецификация

> Навигация: [[Индекс Obsidian]] · [[Сводная база знаний]]
>
> Связанные документы: [[Модель продукта]] · [[Модель тарифов]] · [[Тарифные планы]] · [[ТЗ для Codex - завершение продукта]]

## Статус документа
Этот документ — главный источник правды для продукта.

Если что-то расходится между:
- обсуждением;
- старыми markdown-документами;
- интерфейсом бота;
- кодом;

приоритет у этой спецификации.

---

## Позиционирование продукта
Мы продаём:

**готовый Telegram-бот для beauty-мастера или команды специалистов, который помогает принимать записи, показывать услуги и портфолио, напоминать клиентам о визитах и управлять работой через Telegram.**

Продукт не позиционируется как:
- бот только для маникюра;
- кастомная разработка с нуля под каждый запрос;
- полноценный SaaS на текущем этапе.

Текущая модель:

**продуктовая основа + адаптация под клиента**

---

## Аудитория
Продукт рассчитан на:
- одного beauty-мастера;
- команду специалистов;
- небольшую студию.

Примеры ниш:
- бровист;
- lash-мастер;
- визажист;
- косметолог;
- массажист;
- парикмахер;
- барбер;
- мастер ногтевого сервиса.

Важно:
в подаче продукт универсальный для beauty-сферы и не должен визуально или текстово ощущаться как решение только для ногтей.

---

## Роли в продукте
### `demo_buyer`
Покупатель смотрит демо, проходит клиентский путь, открывает тарифы и выходит на связь.

### `manager`
Менеджер приводит интерес, работает через lead-ссылку, отслеживает лиды и передаёт тёплого клиента владельцу.

### `admin`
Рабочая роль для управления записями, контентом и коммуникациями.

### `owner`
Полная роль: продажи, рабочие разделы, lead-flow, управление администраторами.

---

## Интерфейсный стандарт
### Общий принцип
Продукт работает в логике `inline-first`.

### Home-экраны
У каждой роли есть свой home-экран:
- buyer home;
- manager home;
- admin home;
- owner home.

### Навигация
- `⬅️ Назад` — шаг назад внутри текущего сценария.
- `🏠 В меню` — выход в home текущей роли.

### Reply-клавиатура
Постоянная нижняя клавиатура не используется как основной интерфейс.

### Переходы
Нельзя строить UX через:
- пустые сообщения;
- массовое удаление истории;
- рваную замену экранов.

---

## Buyer / demo слой
### Главная задача
После 2–3 минут в демо человек должен:
- понять, что это за продукт;
- увидеть пользу для себя;
- захотеть обсудить запуск или выбрать тариф.

### Основные разделы buyer
- `🧭 Путь клиента`
- `🗓 Пройти запись`
- `📜 Прайс`
- `📸 Портфолио`
- `💳 Тарифы и запуск`
- `💬 Связаться по запуску`
- `📱 Mini App`

### Обязательные требования
- без nail-only эмодзи;
- без гендерно-узких иконок мастера;
- без перегруженных экранов;
- один экран = одна понятная задача.

---

## Рабочий слой
### Manager
Минимальный рабочий интерфейс:
- home-экран;
- lead-ссылка;
- лиды.

### Admin
Рабочие разделы:
- записи;
- контент;
- коммуникации.

### Owner
Полный рабочий слой:
- всё из admin;
- продажи;
- лиды;
- управление администраторами.

---

## Функциональное ядро продукта
В текущую продуктовую основу входят:
- запись клиентов;
- solo/team режим;
- выбор мастера в team-режиме;
- услуги и цены;
- портфолио;
- карточки мастеров;
- напоминания;
- рабочая админка;
- demo-режим для продажи.

В текущий продукт не входят как обязательная часть:
- Mini App как основной интерфейс;
- self-service SaaS;
- нестандартные интеграции;
- кастомные функции без отдельного согласования.

## Тарифы и цены
### Источник правды для интерфейса и оплаты
В текущем продукте публичные тарифы и платежи зафиксированы в валюте `USD`.

### Текущие тарифы
1. `Solo Start` — `39.00 USD`
2. `Solo Pro` — `49.00 USD`
3. `Team` — `79.00 USD`

### Рекомендуемый тариф
**Solo Pro**

Он должен выделяться в интерфейсе как основной рекомендованный вариант.

### Дополнительные опции
1. `Доп. раунд правок` — `7.00 USD`
2. `+10 фото в портфолио` — `5.00 USD`
3. `Перенос данных` — `15.00 USD`
4. `Второй язык интерфейса` — `20.00 USD`

Если цена, валюта или состав тарифов меняются, изменения должны одновременно попасть в:
- этот документ;
- тарифные markdown-документы;
- интерфейс бота;
- код оплаты.

---

## Наполнение тарифов
### `Solo Start`
Для одного мастера.

Смысл:
- быстрый старт;
- готовая основа;
- часть наполнения клиент делает сам.

### `Solo Pro`
Для одного мастера.

Смысл:
- более полный запуск;
- меньше ручной подготовки со стороны клиента;
- основной рекомендуемый вариант.

### `Team`
Для команды специалистов или студии.

Смысл:
- multi-master формат;
- выбор мастера;
- раздельные услуги, портфолио и записи.

---

## Визуальный язык
### Что использовать
- нейтральные beauty-формулировки;
- спокойные CTA;
- короткие экраны;
- единый emoji-style.

### Что не использовать
- `💅` как основной эмодзи продукта;
- nail-only лексику в универсальных экранах;
- визуальную подачу, где мастер по умолчанию выглядит только как женский персонаж;
- сухие технические экраны в продающем слое.

---

## Продажный тон
Продукт должен звучать как:
- готовое решение;
- понятный запуск;
- адаптация под формат клиента;
- удобство для одного мастера и команды.

Не звучать как:
- конструктор из функций;
- техно-демо;
- бот только для одной ниши.

---

## Правило синхронизации
Перед любым крупным изменением продукта нужно обновить:
1. эту спецификацию;
2. код;
3. продажные документы;
4. тарифные документы.

Иначе снова появится разрыв между тем, что обсуждали, тем, что продаём, и тем, что реально показывается в продукте.
